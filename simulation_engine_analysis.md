# Argus Telemetry Simulator — Engine Analysis

> **Scope:** `src/argus_telemetry_simulator/` — full analysis of every parameter generated, every tunable knob, and how to replicate these outputs in another simulation.

---

## 1. Overview

The simulator produces a **25-channel telemetry frame** (`TelemetryFrame`) on every tick, generated by a layered pipeline:

```
OrbitalElements → propagate_orbit() → ECI position/velocity
                                    ↓
                  attitude_dynamics → quaternion → Euler angles
                                    ↓
                  sensor_models (GNSS, IMU, StarTracker, Magnetometer, Thermal, Power, Comm)
                                    ↓
                  TelemetryFrame (25 channels)
                                    ↓
          FailureInjector → AttackInjector → final frame
```

---

## 2. The 25 Telemetry Channels

Every frame is a `TelemetryFrame` object, serialisable as a `dict` or `np.ndarray[25]`.

| # | Channel Name | Unit | Normal Range | Source |
|---|---|---|---|---|
| 0 | `position_lat` | degrees | −90 → +90 | `eci_to_llh()` + GNSS noise |
| 1 | `position_lon` | degrees | −180 → +180 | `eci_to_llh()` + GNSS noise |
| 2 | `velocity_x` | km/s | ±8 | Keplerian propagation + GNSS noise |
| 3 | `velocity_y` | km/s | ±8 | Keplerian propagation + GNSS noise |
| 4 | `velocity_z` | km/s | ±8 | Keplerian propagation + GNSS noise |
| 5 | `altitude` | m | ~600 km LEO ≈ 600 000 m | `eci_to_llh()` → km × 1 000 |
| 6 | `acceleration_x` | m/s² | ±10 | Gravity + optional drag |
| 7 | `acceleration_y` | m/s² | ±10 | Gravity + optional drag |
| 8 | `acceleration_z` | m/s² | ±10 | Gravity + optional drag |
| 9 | `temperature` | °C | −50 → +80 | `ThermalModel` (converted from K) |
| 10 | `pressure` | Pa | 1 → 101 325 | Barometric formula: `101325 * exp(−alt_km / 8500)` |
| 11 | `humidity` | % | 0 (space) / 50 (below 100 km) | Step model |
| 12 | `battery_level` | % | 0 → 100 | `PowerModel` (SOC × 100 + noise) |
| 13 | `signal_strength` | dBm | −120 → −20 | `CommunicationModel` (FSL path loss, clamped) |
| 14 | `gyro_x` | rad/s | ~0 ± small | `IMUSensor` gyro + bias + drift |
| 15 | `gyro_y` | rad/s | ~0 ± small | `IMUSensor` gyro + bias + drift |
| 16 | `gyro_z` | rad/s | ~0 ± small | `IMUSensor` gyro + bias + drift |
| 17 | `magnetometer_x` | µT | −50 → +50 | `Magnetometer` (Earth dipole model) |
| 18 | `magnetometer_y` | µT | −50 → +50 | `Magnetometer` |
| 19 | `magnetometer_z` | µT | −50 → +50 | `Magnetometer` |
| 20 | `attitude_roll` | degrees | −180 → +180 | `StarTracker` |
| 21 | `attitude_pitch` | degrees | −90 → +90 | `StarTracker` |
| 22 | `attitude_yaw` | degrees | −180 → +180 | `StarTracker` |
| 23 | `angular_velocity` | rad/s | 0 → ~5 | `‖ω‖` from attitude state |
| 24 | `timestamp` | s (Unix) | monotonically increasing | Simulation clock |

> **Array order:** channels 0–23 followed by timestamp at index 24 in the `.to_array()` output.

## 3. Physics Subsystems

### 3.1 Orbital Mechanics (`orbital_mechanics.py`)

Keplerian two-body propagation using **Kepler's equation** (Newton–Raphson iteration, 10 steps).

**Produces:** ECI position `[x, y, z]` in km and velocity `[vx, vy, vz]` in km/s → converted to lat/lon/alt and velocity channels.

**Optional refinement:** Atmospheric drag acceleration using exponential atmosphere model (`Cd=2.2`, `A=1 m²`, `m=500 kg`).

### 3.2 Attitude Dynamics (`attitude_dynamics.py`)

Quaternion kinematics: `q_new = q_old ⊗ exp(0.5 ω Δt)` with unit-inertia assumption.

**Produces:** Roll, Pitch, Yaw (degrees) and the norm of ω → channels 14–23.

Earth magnetic field: simplified dipole model → channels 17–19.

### 3.3 Sensor Models (`sensor_models.py`)

| Sensor Class | Channels it feeds | Key parameters |
|---|---|---|
| `GNSSSensor` | 0–4 | `position_noise_std`, `position_bias`, `position_drift_rate` |
| `IMUSensor` | 6–8, 14–16 | `gyro_noise_std`, `gyro_bias_{x,y,z}`, `accel_noise_std` |
| `StarTracker` | 20–22 | `attitude_noise_std` |
| `Magnetometer` | 17–19 | `mag_noise_std`, `mag_bias_{x,y,z}` |
| `ThermalModel` | 9 | `temperature_noise_std`, `heat_capacity`, `thermal_resistance`, `ambient_temp`, `heat_generation` |
| `PowerModel` | 12 | `initial_state_of_charge`, `power_consumption`, `solar_generation`, `battery_capacity` |
| `CommunicationModel` | 13 | `signal_noise_std`, `signal_strength`, `frequency`, `transmit_power`, `antenna_gain`, `ground_distance` |

## 5. Failure & Attack Injection Modes

### 5.1 Failure Modes (`FailureMode`)

| Mode | Affected Channels | Mechanism |
|---|---|---|
| `battery_degradation` | `battery_level`, (secondary) `signal_strength`, attitude angles | Exponential or linear SOC drain; low battery cascades to attitude noise |
| `sensor_drift` | Any in `affected_sensors` | Linear or configured drift + growing noise over time |
| `sensor_stuck` | Any in `affected_sensors` | Freezes first captured value; no noise afterward |
| `sensor_dead` | Any in `affected_sensors` | severity > 0.8 → zero output; else random spikes |
| `communication_loss` | `signal_strength` | Permanent (−120 dBm) or intermittent cycling |
| `thermal_anomaly` | `temperature`, (secondary) `battery_level`, attitude angles | Random hot/cold spikes; cascades to battery and attitude at extremes |
| `mechanical_wear` | `attitude_roll/pitch/yaw`, `gyro_x/y/z` | Gradually growing attitude noise over 1 h |

### 5.2 Attack Types (`AttackType`)

| Type | Affected Channels | Mechanism |
|---|---|---|
| `gps_spoofing` | `position_lat`, `position_lon`, `altitude`, `velocity_x/y/z` | Ramped position offset with 0.001 Hz sinusoidal drift + stealth jitter |
| `signal_tampering` | `signal_strength` | Random / oscillating / step tampering signal scaled to ±20 dBm |
| `sensor_compromise` | Any in `target_sensors` | Sine / square / sawtooth waveform added as fraction of current value |
| `trajectory_manipulation` | `position_lat`, `position_lon`, `altitude`, `velocity_x/y/z` | Slow time-proportional orbit drift (0.01 × intensity rate) |
| `multi_sensor_corruption` | Multiple sensors simultaneously | Correlated perturbations: noise for position/velocity, oscillation for env, drift for attitude |
| `jamming` | `signal_strength`, any in `target_sensors` | −80 dBm × intensity drop + high-frequency noise on target sensors |

## 7. Adapting Another Simulator to Emit These Parameters

Follow these steps to make any simulation publish telemetry that is compatible with the 25-channel ARGUS format.

### Step 1 — Define the channel schema

Your output each tick must be a flat dict (or 25-element array in the exact order from §2):

```python
TELEMETRY_CHANNELS = [
    'position_lat', 'position_lon',
    'velocity_x', 'velocity_y', 'velocity_z',
    'altitude',
    'acceleration_x', 'acceleration_y', 'acceleration_z',
    'temperature', 'pressure', 'humidity',
    'battery_level', 'signal_strength',
    'gyro_x', 'gyro_y', 'gyro_z',
    'magnetometer_x', 'magnetometer_y', 'magnetometer_z',
    'attitude_roll', 'attitude_pitch', 'attitude_yaw',
    'angular_velocity',
    'timestamp',
]
```

### Step 2 — Map your state variables to channels

| Your variable | → | ARGUS channel | Conversion |
|---|---|---|---|
| GPS lat | → | `position_lat` | degrees, as-is |
| GPS lon | → | `position_lon` | degrees, as-is |
| Velocity North/East/Down | → | `velocity_x/y/z` | convert to km/s; NED → ECI if needed |
| Altitude MSL | → | `altitude` | convert to metres |
| IMU accelerometer | → | `acceleration_x/y/z` | m/s² |
| Onboard temperature | → | `temperature` | Celsius |
| Barometric pressure | → | `pressure` | Pa (use `101325*exp(−alt_m/8500000)` if no sensor) |
| Relative humidity | → | `humidity` | percentage |
| Battery SOC | → | `battery_level` | percentage (0–100) |
| RSSI / link margin | → | `signal_strength` | dBm, clamp to [−120, −20] |
| IMU gyroscope | → | `gyro_x/y/z` | rad/s |
| Compass / mag sensor | → | `magnetometer_x/y/z` | µT |
| Attitude from AHRS | → | `attitude_roll/pitch/yaw` | degrees |
| Gyro magnitude | → | `angular_velocity` | `sqrt(gx²+gy²+gz²)` in rad/s |
| System clock | → | `timestamp` | Unix epoch float (seconds) |

### Step 3 — Wrap your frame in a compatible class

```python
import numpy as np

class TelemetryFrame:
    CHANNELS = [
        'position_lat','position_lon',
        'velocity_x','velocity_y','velocity_z',
        'altitude',
        'acceleration_x','acceleration_y','acceleration_z',
        'temperature','pressure','humidity',
        'battery_level','signal_strength',
        'gyro_x','gyro_y','gyro_z',
        'magnetometer_x','magnetometer_y','magnetometer_z',
        'attitude_roll','attitude_pitch','attitude_yaw',
        'angular_velocity','timestamp',
    ]

    def __init__(self, **kwargs):
        for ch in self.CHANNELS:
            setattr(self, ch, float(kwargs.get(ch, 0.0)))

    def to_dict(self):
        return {ch: getattr(self, ch) for ch in self.CHANNELS}

    def to_array(self):
        return np.array([getattr(self, ch) for ch in self.CHANNELS], dtype=np.float64)
```

### Step 4 — Add sensor noise (optional but recommended)

To match the simulator's realism, add Gaussian noise to each sensor:

```python
import numpy as np

def add_noise(value: float, std: float) -> float:
    return value + np.random.normal(0, std)

# Example noise standard deviations (match GeneratorConfig defaults)
noise_config = {
    'position_lat':    0.001 / 111.0,  # 0.001 km → degrees
    'position_lon':    0.001 / 111.0,
    'velocity_x':      0.0001,
    'velocity_y':      0.0001,
    'velocity_z':      0.0001,
    'attitude_roll':   0.01,
    'attitude_pitch':  0.01,
    'attitude_yaw':    0.01,
    'temperature':     0.1,
    'gyro_x':          0.001,
    'gyro_y':          0.001,
    'gyro_z':          0.001,
    'magnetometer_x':  0.1,
    'magnetometer_y':  0.1,
    'magnetometer_z':  0.1,
    'signal_strength': 0.5,

```

### Step 5 — Inject failures and attacks

Use `FailureInjector` and `AttackInjector` from the ARGUS library directly on your `TelemetryFrame`:

```python
from argus_telemetry_simulator.core.injectors.failure_injector import FailureInjector
from argus_telemetry_simulator.core.injectors.attack_injector import AttackInjector
from argus_telemetry_simulator.core.config.config import FailureConfig, AttackConfig

# Configure failures
failure_cfg = FailureConfig(
    enabled=True,
    failure_mode="battery_degradation",
    start_time=300.0,
    severity=0.7,
    duration=None,
    affected_sensors=["battery_level"],
    degradation_rate=0.02,
)

# Configure attacks
attack_cfg = AttackConfig(
    enabled=True,
    attack_type="gps_spoofing",
    start_time=600.0,
    intensity=0.8,
    duration=300.0,
    ramp_up_time=10.0,
    position_offset_lat=0.05,
    position_offset_lon=0.05,
    altitude_offset=200.0,
)

failure_injector = FailureInjector([failure_cfg])
attack_injector  = AttackInjector([attack_cfg])

# In your simulation loop:
t = 0.0
dt = 0.1  # 10 Hz
while running:
    frame = your_sim.get_frame()           # → TelemetryFrame compatible object
    frame = failure_injector.apply_failures(frame, t)
    frame = attack_injector.apply_attacks(frame, t)
    publish(frame.to_dict())
    t += dt
```

### Step 6 — Validate output

Use the built-in validator:

```python
from argus_telemetry_simulator.utils.validators import validate_telemetry_frame

errors = validate_telemetry_frame(frame.to_dict())
if errors:
    print("Validation errors:", errors)
```

### Step 7 — Stream to ARGUS server

```python
from argus_telemetry_simulator.integrations.argus_client import ArgusClient

client = ArgusClient(
    websocket_url="ws://localhost:8000/ws/stream",
    channel_id="MY-SIM-1",
)

async def stream_loop():
    async with client:
        while running:
            frame = get_next_frame()
            await client.send_frame(frame.to_dict())
```
